# 快速部署指南

## 一键部署（3步完成）

### 步骤 1：解压项目
将项目文件解压到任意目录

### 步骤 2：启动服务
```bash
docker-compose up -d
```

### 步骤 3：创建管理员账号
```bash
docker-compose exec backend python scripts/create_admin.py
```

**完成！** 访问 http://localhost:20800 即可使用。

---

## 详细说明

### 自动初始化

系统会在首次启动时自动：
- ✅ 创建数据库表结构
- ✅ 初始化功能模块（3个）
- ✅ 配置 Dify API 工作流（8个）

**所有 API 配置已硬编码在代码中，无需手动配置！**

### 默认配置

- **前端地址**：http://localhost:20800
- **后端API**：http://localhost:21800
- **数据库**：PostgreSQL（端口 22800）
- **缓存**：Redis（端口 22801）

### 修改 API 配置

如需修改 Dify API 配置，编辑文件：
```
backend/scripts/setup_all_workflows.py
```

然后重启后端：
```bash
docker-compose restart backend
```

## 常见问题

**Q: 服务启动失败？**
A: 检查端口是否被占用，查看日志：`docker-compose logs`

**Q: 前端无法连接后端？**
A: 等待 1-2 分钟让服务完全启动，然后检查：`docker-compose ps`

**Q: 如何查看初始化日志？**
A: `docker-compose logs backend | grep "初始化"`
